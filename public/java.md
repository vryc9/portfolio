# Définition
Java est un langage de programmation orienté objet qui peut être exécuté sur la plupart des plateformes grâce à la machine virtuelle Java (JVM). Dans un contexte professionnel actuel, c’est l’un des langages les plus utilisés pour le développement Backend et la création d'API qui supportent la logique métier des entreprises. Il reste incontournable côté serveur grâce à des écosystèmes puissants comme Spring (et Spring Boot), qui accélèrent et simplifient la création d'applications web modernes. Cependant, Spring Boot possède un concurrent : Quarkus, un framework principalement basé sur les performances avec un démarrage très rapide ainsi qu’une consommation réduite de ressources.

# Réalisations
J'ai abordé pour la première fois le Java durant ma 3e année avec la mise en place d'une API permettant la création de commandes et leur suivi. Pour ce projet, l’API a été développée en JEE sans l’utilisation de framework. Cela m’a permis de comprendre les différents concepts qui se cachent derrière les frameworks modernes, notamment en configurant les servlets ainsi que les EntityManager à la main.

Par la suite, j’ai consolidé mes acquis et suis monté en compétences en travaillant sur plusieurs projets durant mon alternance. J’ai pu mettre en place un outil permettant l'extraction de données provenant d'un webhook GitLab afin d’automatiser la gestion de scripts SQL. Pour ce projet, j’ai utilisé le framework Spring Boot qui facilite la création d’API. J’ai d’abord commencé par configurer les webhooks sur les projets GitLab. J’ai ensuite récupéré les informations envoyées par les webhooks avec les contrôleurs, pour ensuite les envoyer dans un service afin que les données soient extraites et transformées, avant d'être envoyées au front.

# Autocritique et évolutions
Avec les années, j'ai atteint un niveau intermédiaire en Java, principalement dans le développement d’API REST. Aujourd’hui, je suis autonome sur les différents projets sur lesquels je travaille et, lorsqu’il s’agit d’un nouveau projet, j’arrive à comprendre vite la logique métier. Je maitrise les concepts de la POO, le développement d’API REST avec Spring Boot ainsi que l’écriture de test unitaire avec JUnit. 

Ma marge de progression se situe principalement sur deux axes. Le premier axe serait la communication avec d’autres applications avec l’utilisation de Kafka, RabbitMQ ou bien JMS. Au sein de l’entreprise, la communication se fait via JMS, et à ce jour, je n’ai pas encore eu l’opportunité de travailler avec cette technologie. Le second axe de progression serait l’utilisation d’une architecture hexagonale. Je l’expérimente actuellement sur mon projet personnel car je n’ai travaillé jusqu’ici qu’avec des architectures en couches classiques. 

Avec mon expérience, je conseillerais à quelqu’un qui débute Java de commencer sans framework comme j’ai pu le pratiquer au cours de ma formation. Je trouve que cela aide mieux à comprendre ce qui se passe derrière Spring Boot, l’un des framework Java les plus utilisés. Je recommande aussi d’apprendre Maven ainsi que Spring Boot avec une architecture simple. Cela permet d’apprendre le découpage d’une application en couche avec les controllers, service ainsi que le repository et pour la suite passer sur des architectures plus complexes.

Pour mener une veille constante Je consulte régulièrement Baeldung pour les aspects Spring Boot, ainsi que les publications de Philémon Globléhi sur LinkedIn. De plus, je compte monter et consolider mes compétences sur l’infrastructure en micro-services en suivant une formation chez Programming Techie. En parallèle pour découvrir de nouvelles approches, je développe actuellement un outil de suivi financier où j’utilise Spring Boot avec une architecture hexagonale et une infrastructure en micro-services.

Java est pour moi une compétence centrale dans mon profil de développeur fullstack. Pour le futur, je voudrais évoluer dans des grandes entreprises où Java est très souvent utilisé avec Spring Boot, couplé avec Angular. À moyen terme, mon objectif est d'être capable de concevoir et de travailler de façon autonome sur une architecture micro-services complète, depuis la définition des services jusqu'à leur communication et leur déploiement. C'est précisément ce que j'expérimente en ce moment sur mon projet personnel, et que je compte approfondir avec la formation chez Programming Techie. Cette compétence est donc à la fois valorisante pour mon profil aujourd'hui et un axe de développement prioritaire pour les années à venir.
