# Définition
Git est un système de gestion de versions décentralisé (VCS) permettant de suivre l'évolution du code source, de collaborer à plusieurs et de revenir en arrière en cas d'erreur.

# Réalisations
Tout au long de ma formation et de mon alternance, j’ai utilisé la technologie Git sur divers projets mais c’est durant mon alternance que j’ai pu utiliser Git avec une utilisation plus poussée ce qui m’a permis de gagner en compétence. J’ai principalement utilisé Git avec l’outil Gitlab qui permet de stocker des projets sur un dépôt distant.

Dans le cadre d’un projet en entreprise, j'ai configuré des webhooks GitLab pour automatiser les processus. Pour chaque projet, j'ai spécifié les branches cibles et sélectionné l’événement « Push Events ». Ainsi, à chaque fois qu'un collaborateur pousse du code sur le dépôt, mon API reçoit automatiquement les informations des commits. Grâce à cette configuration, l'intégralité du processus de traitement des scripts est devenue automatique.

# Autocritique et évolutions
Au départ, la compréhension globale du fonctionnement a été complexe en raison du nombre important de commandes disponibles. Ma progression a été constante au fil des années. Aujourd’hui je considère avoir atteint un niveau d'autonomie me permettant de résoudre la plupart des problématiques de versioning rencontrées.  Pour l’évolution je souhaite approfondir mes connaissances sur GitLab CI/CD afin de maîtriser l'automatisation complète du déploiement des applications (Pipeline de test, build et déploiement). Pour les développeurs, cette compétence est incontournable car elle permet de travailler plus facilement et permet l'intégration d'outil d'automatisation.
