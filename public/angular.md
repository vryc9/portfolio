# Définition

Angular est un framework open-source développé par Google, conçu pour créer des applications web monopage (SPA). Il est particulièrement adapté aux applications complexes grâce aux nombreux outils qu’il propose. Basé sur TypeScript, un sur-ensemble de JavaScript avec typage fort, il permet un code plus structuré et maintenable.

Angular intègre son propre système de routage, ce qui permet de naviguer entre différentes vues sans recharger la page. L’un de ses atouts est l’intégration native de RxJS, une librairie qui facilite la gestion des données asynchrones de manière réactive. Mais avec l’arrivée des signaux dans Angular 17, la manipulation de la réactivité devient plus simple, et RxJS pourrait être de moins en moins utilisé.

# Réalisation

Durant mon cursus scolaire ou alternance, j'ai été amené à utiliser Angular pour la création d'interface web sur plusieurs projets. Dans le cadre de mon alternance, j'ai créé un outil permettant la gestion d'état de script SQL. Sur ce projet, j’ai pu réaliser une interface complète en partant de zéro. Angular m'a aidé pour toute la conception de l'interface avec des composants ainsi que pour manipuler les données provenant de l'API (Redirection vers la compétence). J’ai aussi géré l’actualisation des données en temps réel avec l’utilisation des observables mise à disposition par la librairie RxJS(lien vers le site), décrit précédemment.

Au cours de ma première année d’alternance, j'ai élaboré un comparateur XML (Redirection vers la réalisation) permettant la comparaison de deux fichiers XML. Comme pour le premier projet, j’ai créé une interface à partir de zéro. Ce projet avait une complexité a été assez élevée, car toutes les modifications apportées sur un fichier XML, devaient être rapportées sur le second fichier. Pour permettre cela, j’ai utilisé les services d’Angular, dans lequel j’ai développé toute la logique. Grâce à ce service, j’ai pu partager cette logique à travers différents composants.

Depuis la rentrée 2024, je travaille sur un tout nouveau projet sur lequel je dois faire la refonte d’un tableau de bord. Ce projet m’a permis de découvrir NgRx, un gestionnaire d’état permettant de diffuser les changements d’état à travers toute l’application. Au départ, cela a été assez complexe d’appréhender cette nouvelle technologie. Pour monter en compétences, je me suis inspiré de code existant, lu des docs ainsi que la documentation et demander de l’aide à mes collègues. Aujourd’hui, le projet ayant bien avancé, j’ai pu manipuler NgRx sous toute ses formes, ce qui me rend 100% autonome.

# Autocritique et évolutions
Aujourd’hui, je pense avoir développé une assez bonne maîtrise d'Angular. Avec les projets que j’ai pu réaliser durant mon cursus ou mon alternance, j’ai pu aborder des concepts de base d’Angular ainsi que des aspects un peu plus avancés sur les dernières années. 
Pour rester informer des dernières nouveautés, je mène une veille constante avec des postes LinkedIn ou encore via le site Dev.to. Pour moi, Angular est une très bonne compétence à avoir, car elle est particulièrement utile pour les grands projets et assez sollicité par les grandes entreprises.
