# Définition

Angular est un framework **open-source** développé par Google, conçu pour créer des applications web monopage (SPA). Il est particulièrement adapté aux applications complexes grâce aux nombreux outils qu’il propose. Basé sur TypeScript, un sur-ensemble de JavaScript avec typage fort, il permet un code plus structuré et maintenable.

Angular intègre son propre système de routage, ce qui permet de naviguer entre différentes vues sans recharger la page. L’un de ses atouts est l’intégration native de RxJS, une librairie qui facilite la gestion des données asynchrones de manière réactive. Mais avec l’arrivée des signaux dans Angular 17, la manipulation de la réactivité devient plus simple, et RxJS pourrait être de moins en moins utilisé.

# Réalisation

Durant mon cursus scolaire ou alternance, j'ai été amené à utiliser Angular pour la création **d'interface web** sur plusieurs projets. Dans le cadre de mon alternance, j'ai créé un outil permettant **la gestion d'état de script SQL**. Sur ce projet, j’ai pu réaliser une interface complète en partant de zéro. Angular m'a aidé pour toute la conception de l'interface avec des composants ainsi que pour manipuler les données provenant de l'API. J’ai aussi géré l’actualisation des données en temps réel avec l’utilisation des observables mise à disposition par la librairie RxJS, décrit précédemment.

Au cours de ma première année d’alternance, j'ai élaboré un **comparateur XML** permettant la comparaison de deux fichiers XML. Comme pour le premier projet, j’ai créé une interface à partir de zéro. Ce projet avait une complexité a été assez élevée, car toutes les modifications apportées sur un fichier XML, devaient être rapportées sur le second fichier. Pour permettre cela, j’ai utilisé les services d’Angular, dans lequel j’ai développé toute la logique. Grâce à ce service, j’ai pu partager cette logique à travers différents composants.

Depuis la rentrée 2024, je travaille sur un tout nouveau projet sur lequel je dois faire la **refonte d’un tableau de bord**. Ce projet m’a permis de découvrir **NgRx**, un gestionnaire d’état permettant de diffuser les changements d’état à travers toute l’application. Au départ, cela a été assez complexe d’appréhender cette nouvelle technologie. Pour monter en compétences, je me suis inspiré de code existant, lu des docs ainsi que la documentation et demander de l’aide à mes collègues. Aujourd’hui, le projet ayant bien avancé, j’ai pu manipuler NgRx sous toute ses formes, ce qui me rend 100% autonome.

# Autocritique et évolutions
Aujourd'hui, je me situe à un **niveau avancé** sur Angular : je maîtrise les concepts fondamentaux (composants, services, routing, injection de dépendances) ainsi que des aspects plus avancés comme les signaux introduits en version 16, la gestion d'état avec NgRx ou la programmation réactive avec RxJS. Je ne me considère cependant pas encore au niveau expert, notamment sur les aspects performance et architecture à grande échelle.

Ma principale **marge de progression** se trouve sur la partie performance avec notamment l’apprentissage du detection **strategy OnPush** qui va devenir une configuration par défaut sur des futures version d’Angular. De plus, un autre axe que je ne maitrise pas serait l’utilisation Server Side Rendering (SSR), avec Angular Universal.

Pour rester informé des dernières nouveautés, je mène une veille constante sur **LinkedIn** en suivant des développeurs expérimentés par exemple Matthieu Riegler ou Gaetan Redin qui possède une newsletter. De plus je suis des articles sur des forums sur divers sujet autour d’Angular sur des sites comme **dev.to ou angular.love**. Pour progresser et apprendre des concepts avancés ou découvrir les nouvelles fonctionnalités, je développe actuellement un **projet personnel permettant de faire un suivi financier**, projet qui utilisent les dernières version d’Angular ainsi que de NgRx. Ce projet me permet de monter en compétence sur des concepts que je n’ai pas l’occasion d’aborder en entreprise. Ce qui me permet d’acquérir des notions d’être opérationnel lorsque ces concepts seront utilisés dans un contexte professionnel. 

Avec le recul, je conseillerais à quelqu’un qui souhaite débuter Angular d’apprendre les fondamentaux avec le système de composant, l’injection de dépendance et l’utilisation des signaux qui vont être de plus en plus utilisés sur des projets futurs. Une fois des bases solides, commencer à appréhender RxJs qui est une librairie assez compliquée à comprendre et à utiliser. Pour moi, Angular est une très bonne compétence à avoir, car elle est particulièrement utile pour les grands projets et assez sollicité par les grandes entreprises.

Angular représente pour moi une compétence très importante dans mon profil de développeur fullstack. Les grandes structures utilisent majoritairement Angular pour leur application complexe, ce qui fait un atout par rapport des framework ou librairie plus conventionnelle comme React ou VueJS .
